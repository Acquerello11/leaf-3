# 📘 คู่มือทางเทคนิค: สถาปัตยกรรมระบบจำแนกใบไม้แบบ Hybrid (SAM + DINOv2-Large)
*เอกสารนี้จัดทำขึ้นเพื่ออธิบายหลักการทำงานและรายละเอียดของอัลกอริทึมเชิงลึกที่ใช้ในโปรเจกต์ (อัปเดตล่าสุด: มิถุนายน 2026)*

---

## 1. ภาพรวมของระบบ (System Overview)
ระบบจำแนกสายพันธุ์ใบไม้ของเราใช้สถาปัตยกรรมแบบ **Hybrid Pipeline** ซึ่งเป็นการนำข้อดีของโมเดล 2 ชนิดที่เก่งคนละด้านมารวมกัน:
1. **โมเดลตัดภาพ (Segmentation Model):** เพื่อกำจัดสัญญาณรบกวน (Noise)
2. **โมเดลสกัดลักษณะเฉพาะ (Vision Foundation Model):** เพื่อดึงลวดลายและรูปทรงของใบไม้

การออกแบบเช่นนี้ช่วยแก้ปัญหา "โมเดลแอบจำฉากหลัง" (Background Bias) ซึ่งเป็นปัญหาใหญ่สุดในการทำ Fine-Grained Image Classification (การแยกแยะสิ่งของที่หน้าตาคล้ายกันมากๆ)

---

## 2. อัลกอริทึมที่ใช้ในแต่ละขั้นตอน (Algorithm Details)

### ✂️ ขั้นที่ 1: การไดคัทและลบฉากหลัง (Auto-Segmentation)
- **ไฟล์ที่เกี่ยวข้อง:** `hybrid_01_segment_leaves.py`
- **อัลกอริทึมเป้าหมาย:** U^2-Net (ผ่านไลบรารี `rembg`) หรือ SAM (Segment Anything Model)
- **หลักการทำงาน:** 
  แทนที่จะส่งภาพถ่ายที่มีทั้ง มือคนจับใบไม้, ท้องฟ้า, หรือผืนดิน เข้าไปให้โมเดล เราจะส่งผ่านโมเดล **U^2-Net** ซึ่งเป็นสถาปัตยกรรม Deep Learning ที่ออกแบบมาเพื่อการตรวจจับวัตถุเด่น (Salient Object Detection) โดยเฉพาะ มันจะสร้าง Alpha Mask เพื่อตัดเอาพิกเซลที่ไม่ใช่ใบไม้ออกให้กลายเป็นความโปร่งใส (Transparent)
- **ทำไมถึงใช้:** เพื่อบังคับให้โมเดลตัวถัดไปสนใจแค่ลวดลายและเส้นใบไม้ 100%

### 🗂️ ขั้นที่ 2: การแบ่งสัดส่วนข้อมูล (Data Splitting Strategy)
- **ไฟล์ที่เกี่ยวข้อง:** `hybrid_02_split_data.py`
- **หลักการทำงาน:**
  แบ่งข้อมูลภาพที่ผ่านการไดคัทแล้วออกเป็น 3 ส่วนแบบสุ่ม (Randomized Split) 
  - **Train (70%):** ใช้สำหรับอัปเดตน้ำหนักของโมเดล
  - **Validation (15%):** ใช้สำหรับจูนค่าพารามิเตอร์ (Hyperparameters) หรือเช็คการ Overfitting 
  - **Test (15%):** ชุดข้อสอบลับที่โมเดลไม่เคยเห็นมาก่อน ใช้สำหรับวัดผลขั้นตอนสุดท้าย
- **ทำไมถึงใช้:** เพื่อให้มั่นใจว่าความแม่นยำที่ได้ เป็นความแม่นยำจริงๆ ไม่ใช่เกิดจากการที่โมเดล "จำข้อสอบได้" (Data Leakage)

### 🧠 ขั้นที่ 3: การเรียนรู้และดึงจุดเด่น (Feature Extraction & Metric Learning)
- **ไฟล์ที่เกี่ยวข้อง:** `hybrid_03_finetune_dino.py`
- **อัลกอริทึมที่ใช้:** SOTA Foundation Model (โหลดผ่าน `transformers` ของ HuggingFace) + Supervised Contrastive Learning
- **หลักการทำงาน:**
  1. **HuggingFace Vision Transformer:** เราดึงโมเดลระดับ State-of-the-Art อย่าง `facebook/dinov2-large` (หรือ DINOv3 ในอนาคต) ส่งตรงจากเซิร์ฟเวอร์ HuggingFace มาทำหน้าที่เป็น "ดวงตา" มันจะแปลงภาพใบไม้นั้นให้กลายเป็น "พิกัดเวกเตอร์ 1024 มิติ"
  2. **SupConLoss (Loss Function):** ทำหน้าที่เป็น "ครูผู้สอน" แทนที่จะสอนโมเดลว่า "รูปนี้คือใบไม้พันธุ์ A" มันจะสอนโมเดลให้ **"ดึงเวกเตอร์ของใบไม้พันธุ์เดียวกันให้เข้ามาใกล้กันในอวกาศสามมิติ และผลักเวกเตอร์ของพันธุ์อื่นให้ออกห่างไปไกลๆ"**
- **ทำไมถึงใช้:** การใช้ Contrastive Learning คู่กับภาพที่ถูกลบฉากหลังแล้ว จะทำให้โมเดลได้เรียนรู้ความแตกต่างระดับไมโคร (เช่น ขอบหยัก, ความห่างของเส้นใบ) ได้ดีกว่าการใช้ Cross-Entropy ธรรมดา

### 📊 ขั้นที่ 4: การประเมินผลและการทำนาย (Inference & Evaluation)
- **ไฟล์ที่เกี่ยวข้อง:** `hybrid_04_evaluate.py`
- **อัลกอริทึมที่ใช้:** k-Nearest Neighbors (k-NN) ควบคู่กับ Cosine Similarity
- **หลักการทำงาน:**
  ระบบจะทำงาน 2 สเตป:
  1. สร้าง **"คลังสมอง (Knowledge Base)"** โดยนำรูป Train (70%) ทั้งหมดมาแปลงเป็นเวกเตอร์ 1024 มิติเก็บไว้
  2. เมื่อมีรูป Test เข้ามา จะแปลงรูป Test เป็นเวกเตอร์ แล้วนำไปวัดมุมความต่าง (Cosine Similarity) กับเวกเตอร์นับพันในคลังสมอง 
  3. กฎ **k=1 (1-Nearest Neighbor):** หากเวกเตอร์ของ Test ไปอยู่ใกล้เวกเตอร์ของใครมากที่สุดในคลังสมอง ระบบจะให้คำตอบเป็นสายพันธุ์ของรูปนั้น
- **ทำไมถึงใช้:** เป็นวิธีทดสอบมาตรฐานที่ตรงไปตรงมาที่สุดสำหรับโมเดลที่เรียนรู้แบบ Metric Learning โดยไม่ต้องสร้าง Classification Head มาครอบทับอีกชั้น
